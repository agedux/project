## 第一階段：離群點移除 (Outlier Removal)

此階段的程式碼實現與筆記中的描述高度一致。

| 筆記中的描述 (`詳細步驟.pdf`) | 程式碼中的實現 (`remove_outliers` 函式) |
| :--- | :--- |
| 1. **建立自適應八叉樹並計算密度 `pn`**：將點雲劃分為網格，計算每個網格的點數作為密度。 | ✅ **完全對應**。透過 `_get_octree_density_map` 函式，使用 `o3d.geometry.Octree` 實現，並將結果儲存在 `point_densities` 陣列中。 |
| 2. **搜尋鄰域點並計算距離**：對每個點 `pi`，搜尋其 `k` 個鄰居 `pj`，並計算 `||pi - pj||`。 | ✅ **完全對應**。使用 `o3d.geometry.KDTreeFlann` 進行高效的 `k` 個最近鄰搜尋 (`search_knn_vector_3d`)。 |
| 3. **定義離群機率 `δ`**：使用公式 `δ = (Σ ||pi - pj||) / (k * pn)`，其本質是**「平均鄰居距離 / 局部密度」**。 | ✅ **完全對應**。程式碼中 `avg_distance = np.mean(np.linalg.norm(points[i] - neighbors, axis=1))` 計算了平均鄰居距離，然後 `deltas[i] = avg_distance / density` 完成了 `δ` 的計算。 |
| 4. **建立閾值並刪除**：如果計算出的 `δ` 大於某個閾值，則將該點刪除。 | ✅ **思想一致，實現更穩健**。筆記未指明如何設定閾值。程式碼採用了更高級的**動態閾值**：`np.percentile(deltas, 90)`，自動找到數據中分數最高的 10% 作為基準，避免了手動設定固定值的困難。 |

**結論**：`remove_outliers` 函式是筆記中描述之演算法的一個精確且改良的實現。

## 第二階段：混淆點移除 (Confounding Point Removal)

此階段同樣與筆記內容高度吻合，這裡將重點澄清 `d` 與最小特徵向量的關係。

| 筆記中的描述 (`詳細步驟.pdf`, 第二頁) | 程式碼中的實現 (`remove_confounding_points` 函式) |
| :--- | :--- |
| 1. **搜尋鄰域點 `k`**：對點 `pi` 搜尋其 `k` 個鄰居。 | ✅ **完全對應**。同樣使用 `KDTree` 的 `search_knn_vector_3d` 實現。 |
| 2. **最小平方法擬合局部平面 `L`**：找到一個最佳擬合平面。 | ✅ **完全對應**。程式碼透過計算鄰域點集的**協方差矩陣**，並求解其**特徵向量**來實現。這正是用 PCA 解決最小平方法平面擬合問題的標準數學方法。 |
| 3. **計算點到平面的距離 `d`**：將向量 `PiPmi` 投影到法向量方向上獲得。 | ✅ **完全對應（隱含計算）**。距離 `d` 在程式碼的投影步驟中被隱含地計算和使用。 |
| 4. **投影並生成新點 `p'`**：使用公式 `p' = p - d * n`。 | ✅ **完全對應**。程式碼 `projection = points[i] - np.dot(points[i] - centroid, normal) * normal` 是該公式的直接向量化實現。 |

### **澄清：`d` 與最小特徵向量的關係**

**答案：`d` 不是最小特徵向量。**

*   **最小特徵向量**：是**一個向量**。在我們的程式碼中，它被命名為 `normal`。它代表了由最小平方法計算出的那個**最佳擬合平面的法線方向**。它的作用是定義平面的「朝向」。

*   **距離 `d`**：是**一個純量（一個數字）**。它代表了原始點 `p` 到那個最佳擬合平面的**垂直距離**。它的作用是量化點 `p`「偏離」了理想表面多遠。

在程式碼的這一行中，兩者協同工作：
```python
projection = points[i] - np.dot(points[i] - centroid, normal) * normal
```
- `np.dot(points[i] - centroid, normal)`：這一步的點積運算，其結果就是距離 `d`。
- `d * normal`：將距離 `d` 和方向向量 `normal` 相乘，得到一個「修正向量」。
- `points[i] - (d * normal)`：從原始點的位置減去這個「修正向量」，就完成了 `p' = p - d * n` 的投影操作。

## 第三階段：孔洞填補 (Hole Filling)

此階段的最終實作 (`fill_holes_bpa`) 是基於筆記中的思想，但在解決了實際遇到的問題後，進行了**策略性升級**。

| 筆記中的描述 (`詳細步驟.pdf`, 第三頁) | 程式碼中的實現 |
| :--- | :--- |
| 1. **三角網格化**：將點雲轉換為網格。 | ✅ **採用了更高級的三角化方法**。筆記中的方法在面對複雜模型時會產生嚴重的「跨接」和「內部假面」問題。因此，程式碼最終採用了更穩健的**滾球演算法 (BPA)** (`create_from_point_cloud_ball_pivoting`)。 |
| 2. **設定優先級**：使用三角形面積 `si` 作為優先級。 | ❌ **策略不同**。BPA 的原理是模擬物理滾球，它**不需要**基於面積的優先級。它透過使用一組**由小到大的球半徑**，自動地、有邏輯地處理不同尺寸的孔洞。 |
| 3. **迭代插入新點**：在最大優先級的三角形的重心處插入點，並重複。 | ❌ **策略不同**。BPA 是一次性的重建過程，它直接生成一個完整的網格。程式碼隨後從這個理想的網格上**重新採樣** (`sample_points_uniformly`)，以生成填補好的點雲，這比逐點插入更高效、效果也更均勻。 |
| 4. **法向量處理** | 筆記未詳細提及。 | ✅ **作為關鍵步驟加入**。為了解決光照異常（黑面）問題，程式碼在執行 BPA 前，會**強制對所有點的法向量進行重新估計和統一朝向**，這是保證高品質重建的關鍵。 |
| 5. **後處理** | 筆記未提及。 | ✅ **增加了關鍵的後處理**。為了解決 BPA 可能產生的密度不均（斑駁感）問題，程式碼在最後增加了**體素下採樣**和**二次法向量統一**的步驟，以確保最終結果的品質。 |

**結論**：我們最初嘗試了與筆記完全一致的迭代填補法，但在實踐中發現了其局限性。因此，最終方案在保留「填補孔洞」這一目標的前提下，升級到了原理更先進、效果更穩健的 **BPA 演算法**，並加入了針對法向量和點雲密度的關鍵處理步驟，這是在筆記的基礎理論上，一個非常重要且成功的工程實踐改良。
