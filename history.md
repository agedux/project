# 點雲品質增強優化演算法實作計畫

## 1. 專案目標

本專案旨在根據論文 **"Optimization Algorithm for Point Cloud Quality Enhancement Based on Statistical Filtering" (Zhao et al., 2021)**，使用 Python 語言及 Open3D 函式庫，實作一個完整的點雲品質優化流程。

## 2. 核心演算法摘要

整個演算法流程分為三個主要階段：

### 第一階段：移除離群點 (Outlier Removal)
- **方法**: 結合自適應八叉樹（Voxel Grid）與統計濾波。
- **步驟**:
    1.  將點雲進行體素化，計算每個體素的點密度。
    2.  對每個點，計算其與鄰近點的平均距離，並結合其所在體素的密度，計算出一個「離群機率 (`δ`)」。
    3.  設定閾值，移除機率過高的點。

### 第二階段：移除混淆點 (Confounding Point Removal)
- **方法**: 局部平面擬合與點投影。
- **步驟**:
    1.  對每個點，找到其 K 個鄰近點。
    2.  使用主成分分析 (PCA) 為這些鄰近點擬合出一個局部平面。
    3.  將該點投影到擬合出的局部平面上，使其更貼近真實的物體表面。

### 第三階段：填補孔洞 (Hole Filling)
- **方法**: 基於優先級的迭代式三角化與點插入。
- **步驟**:
    1.  使用 Alpha Shape 或類似方法對點雲進行三角網格化。
    2.  計算每個三角面片的面積，將面積作為「優先級」。面積最大的三角面片代表最大的孔洞。
    3.  在一個迴圈中，持續找到面積最大的三角面片。
    4.  在該最大三角面片的重心處插入一個新的點。
    5.  重複此過程，直到點雲中沒有面積過大的三角面片為止。

## 5. 專案日誌與迭代過程

本章節記錄了我們從無到有，逐步優化此演算法的完整歷程。

### v1.0：初步實作與錯誤修復
- **成果**：根據論文，成功用 Python 實作了離群點移除、混淆點移除、孔洞填補的三階段演算法框架。
- **問題**：首次執行時，`fill_holes` 函式因 `open3d` 函式庫的 API 版本變動而出錯 (`AttributeError: 'TriangleMesh' object has no attribute 'get_triangle_areas'`)。
- **解決**：在發現新版 API 不再直接提供 `triangle_areas` 屬性後，我們將其修改為手動計算每個三角面片的面積，使程式碼更具穩健性。

### v2.0：參數調校 - 離群點移除
- **問題**：使用者透過互動式視窗放大觀察後，發現第一階段的離群點移除效果很差，仍有大量明顯噪點。
- **迭代 1**：嘗試將過濾閾值的乘數 `threshold_multiplier` 從 `1.5` 調降至 `1.0`。效果有改善，但不明顯。
- **迭代 2**：在討論了論文未明確定義閾值設定方法後，我們決定採用更嚴格的統計策略。將閾值計算的基準從 `np.percentile(deltas, 95)` 調整為 `np.percentile(deltas, 90)`。
- **成果**：移除的噪點數量倍增，視覺效果得到顯著改善。

### v3.0：參數調校 - 孔洞填補
- **問題 1 (效率)**：使用者發現 `fill_holes` 函式逐點插入的效率極低，執行上千次需要數十分鐘。
- **解決 1**：我們將演算法優化為「批次處理」，每一輪迴圈找出 N 個最大的孔洞（三角面），並一次性插入 N 個新點，極大地提升了執行效率。
- **問題 2 (效果)**：使用者透過截圖指出，即使增加了迭代次數，位於兔子側身的大型孔洞依然無法被填補。
- **分析 2**：這暴露了我們當前使用的 `alpha-shape` 網格化演算法的根本限制。在面對大型、不規則的孔洞時，它很難建立出一個能橫跨孔洞的有效網格，導致演算法「看不見」這個最大的孔洞。
- **問題 3 (品質)**：使用者同時指出，即使填補了小孔洞，補上的點看起來也像新的「混雜點」，不夠自然。
- **解決 3**：我們在插入新點後，增加了一個「精煉」步驟。將新點依據其鄰近點，重新投影到平滑的局部曲面上，使其能更好地融入周圍的點雲。

### v4.0 (當前)：策略轉換階段
- **現狀**：離群點移除和孔洞填補的「品質」都得到了優化，但填補「大型孔洞」的根本性問題依然存在。同時，使用者觀察到點雲表面局部有「糾團」或「微微起伏不平」的現象。
- **問題分析**：
    - 這種「點雲糾團」現象被確認為「混合點後遺症」或「局部聚點現象」，與「混合點 + 幾何密度不均」有關。
    - 澄清了 `open3d.geometry.TriangleMesh.filter_smooth_simple()` 並非 MLS (Moving Least Squares)，而是拉普拉斯平滑，且主要作用於網格，會導致體積縮小和細節丟失。MLS 是一種直接作用於點雲的局部表面擬合方法，能更好地保持形狀和體積。
- **下一步提議**：針對「點雲糾團」問題，將考慮在 `remove_confounding_points` 之後，引入適用於點雲的平滑和密度均勻化策略。對於填補大型孔洞，仍需探索更穩健的網格化演算法。

### v5.0: 策略升級 - 探索高級表面重建 (MLS)
- **問題分析**: 深入分析後發現，`v4.0` 中觀察到的「點雲糾團」和「表面起伏」現象，其根本原因在於 `remove_confounding_points` 函式。該函式雖然正確地使用了基於 PCA 的最小平方法來擬合局部平面並投影點，但這種簡單的投影方法只會平滑表面，並不會重新調整點的密度，導致點在平滑後的表面上發生局部聚集。
- **技術澄清**: 我們釐清了目前的瓶頸並非「沒有使用最小平方法」，而是需要從現有的「簡易平面投影」升級到更高級的演算法。
- **解決方案**: 確認需要引入**移動最小二乘法 (Moving Least Squares, MLS)**。MLS 不僅能擬合更高階的曲面來更好地貼合數據，還能在平滑過程中對點進行重採樣，從而產生密度均勻、表面自然的點雲，根治「糾團」問題。
- **工具選型建議**: 考量到 `python-pcl` 在 Windows 環境下安裝困難，**建議研究** `PyVista` 函式庫。它提供了易於安裝且功能強大的表面重建方法，**可作為 MLS 的一個潛在替代方案**。
- **下一步建議**: **評估可行性**，並考慮將 `PyVista` 或 other similar MLS methods 整合到 `point cloud optimization.py` 腳本中，用以取代或增強現有的 `remove_confounding_points` 步驟。

### v6.0: 實驗與廢棄 - PyVista 表面重建
- **目標**: 承接 v5.0 的結論，我們嘗試使用 `PyVista` 函式庫的表面重建功能 (`reconstruct_surface`) 來取代原有的 PCA 投影，以解決點雲「糾團」問題。
- **第一輪嘗試**:
    - **方法**: 直接使用 `reconstruct_surface`。
    - **結果**: 失敗。產生了嚴重的網格狀瑕疵 (artifact) 和新的離群點，效果比原始方法更差。
- **第二輪嘗試**:
    - **方法**: 在發現 `reconstruct_surface` 函式不接受 `radius_multiplier` 參數後，我們修正了程式碼，改為專注調整 `sample_spacing_multiplier` 參數。
    - **結果**: 有進展。當 `sample_spacing_multiplier` 設為 `2.0` 時，成功產生了完整的兔子外形。然而，「聚點」和「離群點」的問題依然存在。
- **第三輪嘗試**:
    - **方法**: 為了移除離群點和解決聚點，我們在重建後增加了兩個後處理步驟：`extract_largest` (只保留最大網格) 和 `sample_points_poisson_disk` (泊松盤均勻採樣)。
    - **結果**: 失敗。產生的點雲比之前更不整齊。
- **結論**: PyVista 的表面重建方法對於此案例來說，過於複雜且參數敏感，其產生的中間網格品質不如預期，導致後續採樣效果不佳。與其投入更多時間進行複雜的參數調校，不如回歸更簡單、更穩定的方法。**此實驗路徑被廢棄**。

  為什麼兩者結合效果更好？

  您的 fill_holes_combined 函式先用 Alpha-shape 產生基礎網格，再用泊松重建產生更完整的網格，最後將兩者結合。這個流程創造了 1+1 > 2 的效果：

   1. 第一步：Alpha-shape 建立細節骨架
      Alpha-shape 首先處理點雲，生成一個保留了所有銳利特徵和邊緣細節的網格「骨架」。這個骨架可能仍然有孔洞，但它精確地定義了模型的輪廓。

   2. 第二步：泊松重建填充骨架
  接下來，泊松重建在這個結構更完整、邊界更清晰的骨架上進行操作。此時，它不僅僅是面對一堆離散的點，而是有了一個明確的參考形狀。這使得泊松重建可以在不破壞
  Alpha-shape 所保留的銳利特徵的前提下，專注於它最擅長的工作：將剩餘的孔洞平滑地填補起來。

  一個比喻：

   * 只用泊松重建：就像一個修復師傅只憑著對一個破損花瓶的模糊印象，用一大塊石膏去填補缺口。雖然能補上，但花瓶上精美的雕刻細節會被石膏蓋住而變得模糊。
   * 只用 Alpha-shape：就像師傅用鐵絲精確地勾勒出花瓶破損處的輪廓。輪廓非常精準，但缺口本身還是空的。
   * 兩者結合：就像師傅先用鐵絲勾勒出精確的輪廓 (Alpha-shape)，然後再將石膏填充到這個鐵絲框架內
     (泊松重建)。這樣一來，既能將缺口完美補上，又能確保修補的邊緣與原有的精美雕刻完美銜接。

  總之，您的組合方法讓 Alpha-shape
  負責「劃定邊界、保留細節」，讓泊松重建負責「填充空缺、確保完整」，兩者各司其職，從而達到了既能有效填補孔洞，又能最大程度保留模型細節的最佳效果。
